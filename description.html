Description

This demo is using the Platform API for Isilon (sometimes referred to as PAPI).
The Platform API allows you to query and even send commands to an Isilon cluster. The commands are sent in a RESTful fashion.
This essentially means that the information is being accessed as if the application were accessing a web page from a webserver (the Isilon cluster being that webserver).

To access we do need to provide a username and password (so that it is secure) although note that in this demo application the username and password given are not well protected (it is a simple test).

The information sent back is in a format called JSON which is a method for encoding an array, which can be decoded and then used in a program (in this case PHP).

The demo uses PHP and JQUERY (PHP doing the work, JQUERY making it act in a nice interactive manner).

Although you can see all of the Source code to highlight the two most important lines from the PHP code:

$encodedcontents = @file_get_contents($service_url,false,$context);

The file_get_contents function downloads the 'web content' from the array using the url given. To see the urls see the examples under the 'view detail' links, these are formed into the $service_url variable. Note the $context variable is just to setup the web authentication as PAPI uses the secure web protocol HTTPS rather than simple HTTP.

$decodedcontents = @json_decode($encodedcontents,true);

The json_decode function takes the raw JSON encoded text that we downloaded in the last command and re-encodes it from JSON text to a useable, native array (or group of variables) so that we can do something with it in PHP.

Although this is a simple '101' demo, to see how PAPI can be used to help create powerful applications for use with Isilon clusters see:
